# FAQ

## Purpose

Answer recurring questions about where information should live and how readers should navigate between the README, source files, and the wiki.

## Audience

Users and contributors who need project-specific guidance rather than math theory.

## Why Use A Wiki Instead Of Only The README?

The README is the first-contact surface. The wiki is for deeper guides, workflows, and examples that would make the README harder to scan.

## Where Should Exact API Details Live?

Code and Zig doc comments are the canonical source. Wiki pages should link to the exact file or module when behavior needs precise wording.

## What Belongs In A Pull Request?

Any wiki source change in this repository should be reviewed in the same pull request as the related code or documentation change.

## How Should New Pages Be Named?

Use the naming rules in [Templates](./Templates.md) so GitHub Wiki page names stay predictable and easy to link.
