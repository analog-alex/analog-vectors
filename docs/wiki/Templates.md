# Templates

## Purpose

Define the standard page structure and naming rules for GitHub Wiki content so new pages stay consistent and easy to review.

## Audience

Maintainers and contributors creating or revising wiki pages.

## Related Links

- [Home](./Home.md)

## Naming Conventions

- Use `Title-Case-With-Dashes` for GitHub Wiki page names.
- Match the page filename to the final wiki page name.
- Prefer short, descriptive names tied to the user task or domain area.
- Reserve underscored filenames such as `_Sidebar.md` for GitHub Wiki navigation files.

## Standard Page Template

Use this structure for new pages:

```md
# Page Title

## Purpose

One short paragraph describing why this page exists.

## Audience

Who should read the page first.

## Related Links

- Link to the README, source file, or neighboring wiki pages.
```

Add deeper sections after the template only when the page needs them.
