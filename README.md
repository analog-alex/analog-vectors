# analog-vectors

`analog-vectors` is a lightweight Zig math library for vectors, matrices, quaternions, geometry, and SIMD-friendly intersection helpers.

It is a good fit for graphics, simulation, tooling, and any project that wants small, explicit math primitives instead of a heavyweight engine dependency.

## Requirements

- Zig `0.15.1` or newer
- Tested with Zig `0.15.2`

## Installation

### Zig package manager

Fetch the package and let Zig fill in the hash:

```bash
zig fetch --save https://github.com/analogAlex/analog-vectors/archive/refs/tags/v0.1.0.tar.gz
```

Then add it to your `build.zig.zon`:

```zig
.dependencies = .{
    .vectors = .{
        .url = "https://github.com/analogAlex/analog-vectors/archive/refs/tags/v0.1.0.tar.gz",
        .hash = "<generated by zig fetch --save>",
    },
},
```

Wire the module into `build.zig`:

```zig
const vectors_dep = b.dependency("vectors", .{
    .target = target,
    .optimize = optimize,
});

exe.root_module.addImport("vectors", vectors_dep.module("vectors"));
```

### Local checkout

If you are developing against a local clone, point the dependency at the checkout:

```zig
.dependencies = .{
    .vectors = .{
        .path = "../analog-vectors",
    },
},
```

## Quick Start

```zig
const std = @import("std");
const vectors = @import("vectors");

const vec2 = vectors.vec2;

pub fn main() void {
    const a = vec2.init(2, 3);
    const b = vec2.unitX();
    const c = vec2.sum(a, b);

    std.debug.print("c = {any}, length = {d}\n", .{ c, vec2.length(c) });
}
```

Common `Vec2` operations:

- `vec2.init(x, y)` for direct construction
- `vec2.sum(lhs, rhs)` and `vec2.sub(lhs, rhs)` for arithmetic
- `vec2.length(v)` and `vec2.normalize(v)` for vector analysis
- `vec2.dot(lhs, rhs)` and `vec2.cross(lhs, rhs)` for products
- `vec2.rotate(v, radians)` for 2D rotation
